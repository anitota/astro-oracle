# Дневен Оракул

Малко астро приложение: дневна таро карта + хороскоп по зодия + астро прогноза с реална лунна фаза.

## Структура на проекта

```
astro-oracle/
├── index.html      ← структурата на страницата
├── style.css       ← целият дизайн
├── script.js       ← логиката (карти, хороскоп, localStorage)
├── data.json       ← всички текстове/карти — редактираш само тук, за да добавиш съдържание
└── README.md
```

Данните (карти, зодии, хороскоп текстове) са в `data.json`, отделени от кода — така можеш да добавяш нови карти, нови фрази или дори нов език, без да пипаш JS логиката.

## Как да го качиш безплатно

### 1. GitHub (препоръчително, за да имаш версии и после да развиваш)

1. Направи си акаунт в **github.com**, ако нямаш.
2. Създай нов repository (например `astro-oracle`) — Public.
3. Качи и четирите файла (index.html, style.css, script.js, data.json) — през бутона **"Add file → Upload files"** в GitHub.
4. Включи **GitHub Pages**: Settings → Pages → Source: `main` branch → Save.
   След минута сайтът е достъпен на `https://твоят-акаунт.github.io/astro-oracle/`.

### 2. Vercel (по-удобно за развитие после)

1. Регистрирай се на **vercel.com** с GitHub акаунта.
2. "Add New Project" → избери repo-то `astro-oracle`.
3. Deploy — готово, получаваш линк като `astro-oracle.vercel.app`.
4. Оттук нататък всеки път, когато качиш промяна в GitHub, сайтът се обновява автоматично.

## Идеи за по-нататъшно развитие

- **Реален astrology API** — вместо генерирани текстове, да дърпаш данни от истински astrology API (напр. позиции на планетите) през `fetch()` в script.js.
- **Потребителски профили** — вместо localStorage (само на едно устройство), да добавиш вход и база данни (напр. Supabase или Firebase), за да пазиш профила между устройства.
- **Дата на раждане** — да пресмяташ реален зодиакален/асцендентен знак по точна дата и час на раждане.
- **Push известия** — напомняне всеки ден да си дръпнеш картата (изисква PWA / service worker).
- **Повече карти** — да добавиш и 56-те карти от Minor Arcana в `data.json`.

## Локално тестване

Тъй като `script.js` зарежда `data.json` през `fetch()`, отварянето на `index.html` директно като файл (`file://`) няма да зареди данните заради browser ограничения. За локален тест пусни:

```bash
python3 -m http.server 8000
```

и отвори `http://localhost:8000` в браузъра.
